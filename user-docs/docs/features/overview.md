# Features Overview

Communities provides a comprehensive suite of features for community resource sharing and collaboration.

## Core Features

### :material-account-group: [Communities](../concepts/communities.md)

The foundation of Communities. Each community is an independent group with its own members, resources, and governance.

**Key Capabilities:**
- Create and configure communities
- Customize trust thresholds
- Set community values and norms
- Manage member access

**Learn More:** [Communities Concept](../concepts/communities.md) | [Creating a Community](../admin/creating-community.md)

---

### :material-handshake: [Trust System](../concepts/trust-system.md)

Peer-to-peer trust system where members earn permissions through community validation.

**Key Capabilities:**
- Award trust to members you trust
- Remove trust when relationships change
- Earn permissions by gaining trust
- Transparent audit trail

**Trust Score** = Number of community members who trust you

**Learn More:** [Understanding Trust](../getting-started/understanding-trust.md) | [Award Trust Guide](../guides/award-trust.md)

---

### :material-account-supervisor: [Members & Permissions](../concepts/members-roles.md)

Dual permission model combining role-based and trust-based access control.

**Key Capabilities:**
- Role-based permissions (admin-assigned)
- Trust-based permissions (community-awarded)
- Multiple paths to access
- Flexible governance models

**Learn More:** [Permissions Concept](../concepts/permissions.md) | [Managing Members](../admin/managing-members.md)

---

## Resource Management

### :material-gift: [Wealth Sharing](wealth-sharing.md)

Share and request resources within your community without monetary exchange.

**Key Capabilities:**
- Share products and services
- Request resources you need
- Message coordination
- Track fulfillment
- Recurrent services

**Wealth Types:**
- **Objects**: Physical items (food, tools, clothing)
- **Services**: Time, skills, labor

**Learn More:** [Wealth Sharing Guide](wealth-sharing.md) | [How to Share](../guides/share-resources.md) | [How to Request](../guides/request-resources.md)

---

### :material-pool: [Pools](pools.md)

Aggregate resources for collective distribution and planning.

**Key Capabilities:**
- Create resource pools
- Accept contributions
- Coordinate distribution
- Mass distribution wizard
- Track needs vs. inventory

**Use Cases:**
- Weekly farmers markets
- Tool libraries
- Emergency supplies
- Collective projects

**Learn More:** [Pools Guide](pools.md) | [Create a Pool](../guides/create-pool.md)

---

### :material-account-supervisor-circle: [Councils](councils.md)

Specialized groups that manage resources and create initiatives.

**Key Capabilities:**
- Form working groups
- Manage resources
- Create initiatives
- Record usage reports
- Track pool consumptions
- Internal trust system

**Council Types:**
- Resource councils (Food, Housing, Tools)
- Domain councils (Garden, Tech, Childcare)
- Project councils (Events, Infrastructure)

**Learn More:** [Councils Guide](councils.md) | [Join a Council](../guides/join-council.md)

---

### :material-clipboard-list: [Needs System](needs-system.md)

Express future resource requirements for community planning.

**Key Capabilities:**
- Express upcoming needs
- View aggregated community needs
- Help producers plan
- Coordinate pool distributions
- Recurring needs

**Need vs. Request:**
- **Need**: Future requirement (planning)
- **Request**: Immediate ask (existing resource)

**Learn More:** [Needs System Guide](needs-system.md)

## Member Capabilities

### :material-chart-line: [Value Contributions](value-contributions.md)

## Conflict Resolution

### :material-scale-balance: [Dispute Resolution](disputes.md)

Transparent mediation system for resolving conflicts.

**Key Capabilities:**
- Create disputes for unfulfilled requests
- Propose mediation
- Work toward resolution
- Privacy controls (open/anonymous)
- Resolution types (closed/open)

**Process:**
1. Create dispute
2. Mediators propose
3. Participants accept mediator
4. Work toward resolution
5. Mediator creates resolution
6. Participants accept

**Learn More:** [Disputes Guide](disputes.md)

## Feature Combinations

Features work together to create powerful workflows:

### Resource Planning Workflow

1. **Needs** - Members express future requirements
2. **Pools** - Councils create pools to meet needs
3. **Wealth Sharing** - Members contribute to pools
4. **Distribution** - Pool managers distribute based on needs
5. **Value Recognition** - Contributions are tracked and recognized

### Governance Workflow

1. **Forum** - Discuss ideas and proposals
2. **Polls** - Vote on options
3. **Initiatives** - Councils create action plans
4. **Reports** - Track progress
5. **Trust** - Build credibility through follow-through

### Onboarding Workflow

1. **Invite** - Join via invitation link
2. **Profile** - Set up profile with skills
3. **Forum** - Participate in discussions
4. **Trust** - Earn trust from members
5. **Sharing** - Begin sharing resources
6. **Councils** - Join working groups
